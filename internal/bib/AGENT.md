# internal/bib

Handle `.bib` file processing.

## Pipeline (`validate.go`)

Entry point: `ProcessBibFiles(bibFiles, auxDir, abbreviateJournals, braceTitles, ieeeFormat, maxAuthors, abbreviateFirstName, urlFromDOI)`.

For each bib file:
1. Parse â†’ `[]Item` (entries + raw chunks)
2. `assignCanonicalKeys` (first pass)
3. Each unseen entry: `validateEntry` â†’ Crossref/arXiv fix (arXiv entries with DOI redirect to Crossref); Crossref titles cleaned via `cleanCrossrefTitle` (MathMLâ†’LaTeX `$...$`, face markupâ†’LaTeX commands, remaining XML tags stripped); store pending cache
4. `normalizeEntryFields` â€” drop disallowed fields, resolve synonyms, derive url from doi
5. If `ieeeFormat` and `@misc` arXiv: `transformArxivMiscToUnpublished` â€” typeâ†’`unpublished`, drop eprint/archiveprefix/primaryclass, add `note = {[arXiv preprint \href{...}{arXiv:<id>}]}`
6. Format author (truncate to `maxAuthors` if > 0, append `and others`; abbrev first name per `abbreviateFirstName`)
7. Strip non-escaped braces from title (`stripNonEscapedBraces`)
8. If `braceTitles` or `ieeeFormat`: wrap title as `{{inner}}` (idempotent â€” step 7 norm first)
9. `warnMissingFields` â€” print `[bib] <key>: missing mandatory fields: ...`
10. `ensureArticleOptionalFields` â€” add blank `{}` for `volume`, `number`, `pages` on `@article`
11. `sortedFields` â€” reorder to canonical field order
12. For newly validated entries (crossref/arxiv, not cached): compare final field values against pre-processing originals; print `[bib] <key>: reformatted <fields>` for fields that actually changed
13. `assignCanonicalKeys` (second pass â€” keys may shift after Crossref fix)
14. Flush pending cache under final canonical keys
15. Rewrite file if changed

## Single-entry insertion from ID (`validate.go`)

Entry point: `AddEntryFromID(id, auxDir string) (key string, err error)`.

Adds one entry to cache from a raw DOI or arXiv identifier (no bib file needed).

- `normalizeDOI(s)` â€” strips `doi.org/` prefixes, returns bare DOI if starts with `10.`, else `""`
- `normalizeArxivID(s)` â€” matches `reArxivBare` (bare form) or `reArxivURL` (full URL), returns bare ID or `""`
- **DOI path**: dedup via `Fields["doi"]` (case-insensitive); calls `queryCrossref`; entry type set from Crossref `type` field (defaults to `@article`)
- **arXiv path**: dedup via `Fields["eprint"]`; calls `queryArxiv`; if DOI returned, dedup by DOI then redirect to `queryCrossref` (becomes `@article`, `source: "crossref"`); on Crossref failure, falls back to `@misc` with `archiveprefix: {arXiv}`
- Key generated by `GenerateKey`; collision-disambiguated by `disambiguateKey` (appends `a`/`b`/â€¦ suffix)
- Returns `ErrUnrecognizedID` (sentinel `fmt.Errorf`) when format unrecognized â€” caller can check `err == bib.ErrUnrecognizedID`

## Cache allocation (`validate.go`)

Entry point: `AllocateCacheEntries(bibFiles, auxDir) (int, map[string]string, error)`.

Returns count of newly cached entries and a renames map (old key â†’ new canonical key). Parses each bib file and seeds `.el/bib.json` with any entries not yet cached. Parse-only: does NOT rewrite files or run full normalization pipeline. Used by `el init` and `el parsebib` to pre-populate cache without compile.

Deduplication:
- **Entries with DOI**: matched by DOI (case-insensitive). Crossref-validated if new.
- **Entries with arXiv ID**: matched by eprint field (case-insensitive). arXiv-validated if new.
- **No-ID entries**: matched by canonical cite key. Added as `source: "no-id"` placeholder if new.

For new DOI/arXiv entries, runs `validateEntry` and stores full field snapshot like `processBibFile` does. Does not apply config transforms (abbreviateJournals, braceTitles, etc.) â€” those are applied at compile time.

## Bib generation from cache (`validate.go`)

Entry point: `WriteBibFromCache(path, citeKeys, auxDir, abbreviateJournals, braceTitles, ieeeFormat, maxAuthors, abbreviateFirstName, urlFromDOI)`.

Reconstructs entries from cache for given cite keys and writes them to `path` (typically `bibliography.bib`) with all config transforms applied. Called by `el compile` after pass 1 to produce bib file from cache. Only cited keys written.

## Renames persistence (`validate.go`)

- `LoadRenames(auxDir) map[string]string` â€” load `.el/renames.json` (old key â†’ new key)
- `SaveRenames(auxDir, renames)` â€” persist renames map
- `ClearRenames(auxDir)` â€” delete `.el/renames.json`

Produced by `AllocateCacheEntries` when canonical key differs from bib file key. Consumed by `el compile` step 4 to rewrite `\cite{}` before bibliography generation.

## Key generation (`key.go`)

Canonical key: `{LastName}{Year}{Title}` â€” CamelCase, LaTeX accentsâ†’ASCII, math stripped.
`@unpublished` with no year: `{LastName}{Title}`.
Fallback to existing key if component missing.
Disambiguate collisions with `a`, `b`, `c`, â€¦ suffix.

## Entry type specs

Keep `entrySpecs` in `validate.go` and `canonicalOrder` in `format.go` in sync.

| Type | Mandatory | Allowed (= field order) |
|---|---|---|
| `@article` | author, year, title, journal, doi, url | + volume, number, pages |
| `@book` | author, year, title, publisher | + address, doi, url |
| `@incollection` | author, year, title, booktitle, publisher | + address, pages, doi, url |
| `@inproceedings` / `@conference` | author, year, title, booktitle, doi, url | + pages |
| `@phdthesis` / `@mastersthesis` | author, year, title, school, url | + doi |
| `@techreport` | author, year, title, institution, url | + doi |
| `@misc` (base) | author, year, title, url | + doi |
| `@misc` (arXiv) | author, year, title, eprint, archiveprefix | + primaryclass |
| `@unpublished` | author, title, note | + year, doi, url |

### Special rules

- `@article`: `issue` = synonym for `number`; `volume`, `number`, `pages` always emitted (blank `{}` if absent)
- `@misc` arXiv detection: `eprint` + `archiveprefix`/`eprinttype = arXiv` (case-insensitive), or `url` matching `arxiv.org/abs/â€¦`; `archiveprefix` always normalised to `{arXiv}`
- All types with `doi` + `url` allowed: `url` derived from `doi` as `https://doi.org/{doi}` if absent (not for arXiv misc); when `urlFromDOI=true`, replaces existing url too
- No-id warn suppressed for types where `doi` not mandatory

## Validation sources

- **Crossref**: corrects title, author, journal/booktitle (full name), year, volume, number, pages, doi; title cleaned via `cleanCrossrefTitle` (converts MathML to `$...$` and Crossref face markup `<i>/<sub>/<sup>/...` to LaTeX commands); maps `type` to BibTeX entry type (`journal-article`â†’`@article`, `proceedings-article`â†’`@inproceedings`, `book-chapter`â†’`@incollection`, `book`/`monograph`/`edited-book`â†’`@book`, `report`â†’`@techreport`, `dissertation`â†’`@phdthesis`; unknown types leave entry type unchanged); `container-title` maps to `journal` or `booktitle` based on mapped type; `@article` journal â†’ `AbbreviateISO4` on live entry when `abbreviateJournals=true`
- **arXiv**: corrects title, author, year; if `<arxiv:doi>` present in API response, redirects to Crossref validation (entry becomes `@article`, `source: "crossref"`); falls back to arXiv-only on Crossref failure
- Cache in `.el/bib.json` under final canonical key; stores entry `Type`, all allowed fields as snapshot (pre-config-transform: full journal, raw authors, pre-brace title, raw url from bib file)
- On subsequent compiles, all cached fields re-applied to entry before pipeline; journal re-abbreviated if `abbreviateJournals=true`; old-format crossref/arxiv entries (nil Fields) evicted and re-validated

## ISO 4 abbreviation (`iso4.go`)

`AbbreviateISO4(title string) string` â€” abbrev journal title per ISO 4.

- LTWA data in `ltwa.tsv` (37,670 entries, non-empty only); parse once via `sync.Once`
- Lookup: exact first, then longest-prefix (LTWA entries ending `-` = prefix patterns)
- Language priority when multiple entries match: `"multiple languages"` > `"english"` > first record
- First letter of abbrev mirrors original case
- Stop words (articles, prepositions, conjunctions, EN/FR/DE/IT/ES/LA) omitted
- Single-word titles returned unchanged
- Trailing parentheticals stripped before processing: `"Lancet (London)"` â†’ `"Lancet"`
- Hyphenated tokens split on `-`, each segment processed, rejoined
